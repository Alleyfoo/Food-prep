## Known issues / limitations
- The UI builds an in-memory DB cached per session (`@st.cache_resource`); it
  does not watch the YAML for edits. Restart streamlit after curating YAML.
- `pairings.ingredient_id` is the FILLER, not the target; cabbage pairings are
  reached via `works_best_with_transformation_id` (how `scout_rows` and the
  round-6/7 tests resolve them). A future refactor could add an explicit
  `target_ingredient_id` column.
- `risks` is surfaced in the Ingredient/Component Explorer cards; Scout tab
  shows the disclaimer; Plate Balance shows leans-heavy/dry warnings. No
  per-component risk aggregation yet.
- The branch "Try" line now excludes experimental fillers (Cook mode). If a
  future "show me everything" view is wanted, pass
  `include_experimental=True` to `fillers_by_role`.
- `curated_role_fit` is still only annotated on 2 pairings — highest-value
  future task per the checkpoint.
