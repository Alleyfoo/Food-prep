## What was completed this session (round 12 — Markdown export)
A third surface over the same computed dicts: `query` computes → `ui` renders
HTML → `cli` renders text → `export` renders Markdown. No new ontology, no
engine logic — `export.py` only serializes the dicts the UI/CLI already use.
The Round-11 available-items partition flows into Markdown too.

### New module `src/foodprep/export.py`
Two layers (mirrors the plan's "same truth, different surface" rule):
- **Pure renderers** (no `conn`, used by UI + tests):
  `render_branch_markdown(card, partition=None)`,
  `render_component_markdown(d, partition=None)`,
  `render_plate_markdown(r)`, `render_scout_markdown(rows)`.
- **High-level helpers** (take `conn`, used by CLI): `branch_markdown`,
  `component_markdown`, `plate_markdown`, `scout_markdown`. Return `None`
  on a miss (branch/component) so the CLI prints a clean not-found.
- `SCOUT_DISCLAIMER` (verbatim from the UI Scout tab) + `FOOTER_LINES`
  ("## Notes — Generated from local Ingredient Foundry data…") constants.
- **Suggested next move is mechanical** (user-confirmed): a templated line
  read only from `available_now` + `covered_roles` + gaps — never synthesized.
- Shared helpers `_add_section`, `_partition_blocks`, `_add_tags/risks/missing/
  uses/next_moves` keep branch & component renders DRY. The partition render
  (Available now / Missing but useful / No match) is shared by branch, component,
  and plate via `_partition_blocks`.
- Plate render always shows "Missing — hard gaps" + "May want more", then the
  partition (when present), then Risks/Avoid (leans_heavy/dry), then
  "No balance profile for", then "Suggested next move", then the footer.

## Important constraints (do not violate)
- **No vision.** The model cannot read images. Verify screenshots with PIL
  (`Image.open(p).verify()`). Re-capture via `scripts/capture_screenshots.py`
  only. Never visually inspect a PNG.
- **Hedgerow rule.** A new full ingredient must model a genuinely NEW food
  state. Broccoli passed via `steam` + `tender_crisp`. Carrots/mushrooms
  deferred; identify the new state first.
- **Don't call it "pantry".** Use `available_*` / `.avail-*` everywhere.
- **CI-guarded files.** Never stage without explicit user confirmation:
  `.github/workflows/ci-full.yml`, `scripts/ci_build_smoke.py`,
  `scripts/smoke/ci_gui_smoke.py`,
  `scripts/checks/check_update_correctness_signatures.py`.
- **Markdown export guardrail.** `export.py` must only serialize computed
  dicts — never rebuild an answer from strings or add engine logic. The
  "Suggested next move" line is deliberately mechanical (reads the partition,
  invents nothing).
