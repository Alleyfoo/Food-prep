## What was completed this session (round 11 — option A, available filter)
A Cook-only filter: a single multiselect above the tabs lets the user pick
the ingredients on hand; the engine partitions each branch's curated fillers
into **Available now / Missing but useful / No match**. Deliberately NOT a
pantry, inventory, or shopping system — the user warned that "pantry" opens
swamp doors, so the feature is named "available ingredients" / "What do I have
right now?" everywhere, including internal symbols (`available_selector`,
`.avail-strip`). The only remaining `pantry` strings are pre-existing
culinary vocabulary in `tomato.yaml` (the `pantry_base` dish context) and one
negation in the docstring ("Not a pantry system") that reinforces the rule.

### Query layer (`src/foodprep/query.py`)
- `_partition_by_available(by_role_names, available, known)` — pure helper:
  splits a `{role: [filler_name]}` map (Cook-only) against on-hand ingredients
  into `available_now` / `missing_but_useful` / `unknown_items` /
  `no_match_known` / `covered_roles`. Honest: never invents fillers; unknown
  selections are reported, not dropped.
- `available_filter(conn, transformation_id, available_items)` — partitions a
  single branch via Cook `fillers_by_role` (no experimental). None/[] = current
  behaviour.
- `plate_balance_detail(conn, text, available_items=None)` — when
  `available_items` is not None, ALSO returns the partition over the plate's
  `target_gap + flagged_more` roles (Cook fillers via `_fillers_for_role`).
  None keeps the current path (no partition keys in the result).

### UI (`src/foodprep/ui/streamlit_app.py`)
- `available_selector()` — multiselect of `query.ingredients_list` above the
  tabs, with the "What do I have right now?" label + hint strip.
- `available_partition_html(part)` — renders the three buckets as card rows;
  returns '' when empty so the plain Try view shows through.
- `branch_card_html(d, available=None)` — when a non-empty partition is
  passed, it replaces the "Try" row with the partition.
- `tab_ingredient_explorer(available_items)` / `tab_component_explorer(...)` /
  `tab_plate_balance(...)` — each takes `available_items`, calls
  `query.available_filter` / `plate_balance_detail(..., available_items=...)`,
  and renders the partition. Empty selection → `None` → current render.
- Layout: `topbar(); available_items = available_selector(); st.tabs([...])`;
  Tab 1/2/3 receive `available_items`, Tab 4/5 do not.

## Important constraints (do not violate)
- **No vision.** The model in this environment cannot read images. Never ask
  it to inspect a PNG — it crashes/hangs. Verify screenshots with PIL
  (`Image.open(p).verify()`). Screenshots are re-captured via Playwright only.

  state, not "roast it like cabbage." Broccoli passed via `steam` +
