## Issues encountered and resolved

- The journey reader initially referenced a nonexistent newline helper; it now
  uses the query module's existing `_split_list`.
- The existing `foodprep.sqlite` predates journey tables. CLI table detection
  now handles that safely and tells the user to rebuild.
- Seven legacy tests depended on the exact phrase “missing for a balanced
  plate.” That wording remains for the default profile while contextual profiles
  use “required for this destination.”
- The route catalogue lacks `sesame`, `ginger`, and `scallion`. They were not
  invented merely to satisfy roadmap route names. Initial routes use existing
  curated ingredients instead.
