## What was completed this session (round 4)
- **`query.plate_balance(conn, text)`** — the core engine. Recognises each plate
  item as `profile` / `ingredient` / `unknown`, aggregates provided roles
  (profiles + ingredient base_roles) and missing_risks (profiles), both run
  through `_canon_role` (ROLE_CANON + `MISSING_TERM_TO_ROLE`: sauce→hydration,
  fresh_side/freshness→herb). Computes plate heaviness/dryness (mean of profile
  scores) with qualitative reads + lean warnings.
- **Two-tier missing logic** (the key design call):
  - `target_gap = TARGET_ROLES - provided` → "missing for a balanced plate:"
    (hard gaps, fillers grouped by role).
  - `flagged_more = risk_roles - target_gap` → "also flagged by item profiles
    (may want more):" (roles provided but still flagged — e.g. roasted_tomato
    provides acid yet lists acid as a risk). Prevents "add protein to a gratin
    that already has protein".
- **Helpers**: `_recognise_plate_item`, `_match_ingredient` (longest name/alias
  wins), `_match_profile` now strips `_component`/`_base` suffixes, `_canon_role`,
  `_heaviness_label` (heavy≥4/rich≥3/balanced≥2/light), `_dryness_label`
  (dry≥3.5/medium≥2/moist), `_fillers_for_role(..., include_experimental=False)`.
- **`_plate_phrases`** handles `i have / i've got / i got / balance / plate of /
  plate / with` prefixes with a bare-text fallback for CLI input.
- **Routing**: `answer()` routes to `plate_balance` on `balance` / `plate of` /
  `plate balance` / (missing+≥2 items). Tightened so bare "plate" does not
  false-trigger on incidental "on a plate". `meal_repair` kept as backward-compat
  wrapper → `plate_balance`.
- **Cook vs Scout separation**: `plate_balance` labelled "Cook mode", excludes
  experimental pairings (rye_crumbs absent). `scout` labelled "Scout /
  experimental", surfaces them. Verified by `test_cook_and_scout_are_separate`.
- **CLI**: `foodprep plate "<items>"` subparser + `cmd_plate` (checks
  component_profiles populated first).
- **Honesty**: unknown items named not guessed; known-but-profile-less
  ingredients (onion) recognised as ingredients with base_roles AND warned for
  no balance profile (softer than "unknown"); empty input → "Name the plate
  items".
- **Tests**: 52 passing (was 42). +11 plate-balance. Round-3 honesty test
  (`test_meal_repair_boiled_potato_onion_*`) updated: onion is now "(ingredient)"
  not "unknown".
- **Eval doc**: round-4 section appended to `docs/test_outputs/tomato_manual_eval.md`.

## Known issues / limitations
- `_plate_phrases` is regex-based; unusual phrasings may not extract items.
  Covered prefixes: `i have / i've got / i got / balance / plate of / plate /
  with`, plus bare-text fallback.
- Heaviness/dryness only come from profiles — ingredient items (onion) with no
  profile contribute no score (honest: they have no balance data).
- Scout pairings still hand-curated, not corpus-rarity-derived (future work).
- 22 pairings resolve to corpus 0 (olive_oil, pasta_water, etc.) — the override
  map `CANONICAL_TO_CORPUS` can be extended.
