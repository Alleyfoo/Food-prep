## Important issue resolved

The first real-corpus smoke test incorrectly labelled the specialized candidates
as common because legacy backfill aliases intentionally map:

- `brown_butter` → generic `butter`;
- `lingonberry_vinegar` → generic `vinegar`.

Those broad mappings are useful for legacy pairing evidence but invalid for
novelty coverage. Novelty now uses strict identity-level resolution. Against the
local 45,773-recipe CulinaryDB snapshot, both candidates correctly report:

`insufficient_coverage`

The corpus does not contain exact brown-butter or lingonberry-vinegar entities,
so it cannot support either `not_observed` or a frequency class. This is an
honest coverage limitation, not a zero-occurrence claim.
